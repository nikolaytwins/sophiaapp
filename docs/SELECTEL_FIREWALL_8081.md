# Подробно: пункт 1 — открыть порт 8081 (Selectel + сервер)

Цель: с **интернета** (в т.ч. с iPhone по LTE/Wi‑Fi) можно было подключиться к **Metro / Expo** на твоём сервере по адресу  
`http://ТВОЙ_ПУБЛИЧНЫЙ_IP:8081`.

Если порт закрыт на уровне **облака** или **самой виртуалки**, Expo Go покажет: *Could not connect to development server*.

---

## Часть A. Облачный фаервол Selectel (обязательно проверить)

У Selectel доступ к серверу из интернета часто фильтруется **не только** внутри Linux, а **в панели** (группы безопасности, сетевой ACL, «фаервол» и т.п.). Названия разделов могут чуть отличаться в зависимости от продукта (облако VPC / выделенный сервер / старая панель).

### Что нужно по смыслу

Одно правило **входящее (ingress / inbound)**:

| Параметр        | Значение |
|-----------------|----------|
| Протокол        | **TCP**  |
| Порт            | **8081** (или диапазон `8081–8081`) |
| Направление     | **Входящий** к этому серверу |
| Источник (откуда) | Чаще всего **любой** → `0.0.0.0/0` (весь интернет). Если хочешь только свой IP — укажи свой `/32`. |
| Действие        | **Разрешить** |

**Не путать** с исходящим (egress) — нам нужен именно **вход** на сервер.

### Где это искать в панели (ориентиры)

1. Зайди в **личный кабинет Selectel** под своим аккаунтом.
2. Найди свой **сервер / виртуальную машину** (список серверов, инстансов, ВМ).
3. Открой **настройки сети** у этой машины. Возможные названия:
   - **Группы безопасности** / **Security groups**
   - **Фаервол** / **Firewall** / **Правила ACL**
   - **Сеть** → подсеть / роутер → привязанная группа безопасности
4. Выбери группу, которая **привязана к интерфейсу** сервера с **публичным IP** (тем же, что `178.72.x.x` или что показывает `curl ifconfig.me` с сервера).
5. Добавь **новое правило** по таблице выше.
6. **Сохрани** изменения. Иногда правило применяется с задержкой 1–2 минуты.

### Дополнительно (если Expo попросит в логах)

Иногда для отладки нужны ещё порты — добавь по той же схеме, если увидишь в логах Metro/Expo:

- **19000/tcp**
- **19001/tcp**

Для минимального сценария «только Expo Go + Metro» обычно достаточно **8081**.

### Если в панели «ничего не понятно»

- В **документации Selectel** в поиске набери: *группа безопасности*, *firewall*, *правила входящего трафика* — открой актуальную статью под **твой тип** продукта (облако vs выделенный сервер).
- В **тикет поддержки** можно спросить одной фразой: *«Нужно разрешить входящий TCP 8081 на публичный IP сервера X для dev-сервера. В каком разделе панели добавить правило?»*

## Часть B. Фаервол на самой Linux‑ВМ (Ubuntu и т.п.)

Даже если в Selectel всё открыто, **ufw** на сервере может резать порт.

### Проверить, включён ли ufw

По SSH на сервере:

```bash
sudo ufw status
```

Если видишь `Status: active` — нужно правило.

### Разрешить 8081

```bash
sudo ufw allow 8081/tcp comment 'Expo Metro'
sudo ufw reload
sudo ufw status numbered
```

Убедись, что в списке есть разрешение **8081/tcp** **IN** **Anywhere** (или твой нужный источник).

### Если ufw выключен

`Status: inactive` — этот слой обычно не мешает; главное — облако (часть A).

## Часть C. Убедиться, что Metro слушает «для всех интерфейсов»

На сервере:

```bash
cd /root/.openclaw/workspace/sophia-os
npm run check:mobile
```

Или вручную:

```bash
ss -tlnp | grep 8081
```

Хорошо, если есть что‑то вроде `0.0.0.0:8081` или `*:8081`.  
Плохо для телефона из интернета, если **только** `127.0.0.1:8081` — тогда Metro не принимает внешние подключения (нужен запуск через `npm run start:lan`, не только локальный режим).

Запуск:

```bash
npm run start:lan
```

## Часть D. Проверка «снаружи»

1. **С iPhone в Safari:**  
   `http://ТВОЙ_ПУБЛИЧНЫЙ_IP:8081`  
   Должна открыться хотя бы страница/ответ от Metro (не обязательно красивый UI — важно, что **соединение есть**).

2. **С другого компьютера** (не с самого сервера):

   ```bash
   curl -I --max-time 5 http://ТВОЙ_IP:8081
   ```

   Если **таймаут** или **connection refused** — снова часть A (облако) или B (ufw), или Metro не запущен.

## Краткий чеклист пункта 1

- [ ] В панели Selectel добавлено **входящее** правило **TCP 8081** на сервер с публичным IP.
- [ ] При активном **ufw** выполнено `ufw allow 8081/tcp`.
- [ ] На сервере запущен **`npm run start:lan`**, в `ss` видно прослушивание не только на `127.0.0.1`.
- [ ] С телефона открывается **`http://IP:8081`** в Safari.

После этого в Expo Go: `exp://IP:8081`.

Если всё равно не грузится: **[MOBILE_TROUBLESHOOTING.md](./MOBILE_TROUBLESHOOTING.md)** и `npm run check:mobile` на сервере.
