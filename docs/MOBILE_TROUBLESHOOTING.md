# Мобилка не грузится — проверка по шагам (после открытия фаервола)

Иди **сверху вниз**. Как только шаг «ломается» — чини его, дальше можно не идти.

---

## Шаг 1 — Metro вообще запущен?

На сервере в SSH:

```bash
cd /root/.openclaw/workspace/sophia-os
npm run start:lan
```

Должна висеть сессия с логом Metro, строка вроде **Waiting on http://localhost:8081** и **`Packager host для QR: …`**.

Если процесс упал или ты вышел из SSH без `tmux`/`screen` — Metro мог остановиться. Запусти снова или используй:

```bash
tmux new -s expo
npm run start:lan
```

---

## Шаг 2 — порт слушает «весь мир», а не только localhost?

На сервере:

```bash
npm run check:mobile
```

или вручную:

```bash
ss -tlnp | grep 8081
```

**Нужно:** `0.0.0.0:8081` или `*:8081`.  
**Плохо:** только `127.0.0.1:8081` — с телефона из интернета не достучишься. Тогда запускай именно **`npm run start:lan`**, не «только веб» в режиме, который биндится на localhost.

---

## Шаг 3 — с самого сервера порт отвечает?

```bash
curl -I --max-time 5 http://127.0.0.1:8081
```

Должен быть **не** таймаут: хоть `200`, `404`, `301` — главное, что **соединение есть**.

Если нет — Metro не тот порт / не запущен (вернись к шагу 1).

---

## Шаг 4 — фаервол на самой Linux‑ВМ (ufw)

Облако ты открыл — иногда режет **ufw** на сервере:

```bash
sudo ufw status
```

Если **active** — должно быть разрешено:

```bash
sudo ufw allow 8081/tcp
sudo ufw reload
```

---

## Шаг 5 — с телефона (Safari, не Expo Go)

На iPhone открой в **Safari**:

```text
http://ТВОЙ_ПУБЛИЧНЫЙ_IP:8081
```

(IP тот же, что в `Packager host для QR` / вывод `check:mobile`.)

| Результат | Что значит |
|-----------|------------|
| Страница грузится / что-то от Metro | Сеть и порт **ок** → проблема скорее в **URL в Expo Go** (`exp://IP:8081` без опечаток). |
| Долго висит и ошибка / не открывается | До сервера **не доходит**: ещё раз **правило в облаке** (правильный IP сервера, TCP **8081**, inbound), или **провайдер/антивирус**, или Metro слушает не тот интерфейс. |
| Работает только в Wi‑Fi дома, не в LTE | На VPS обычно не так; если сервер дома — возможен **CGNAT** у оператора → смотри **tunnel** (`npm run start:tunnel`). |

---

## Шаг 6 — Expo Go: правильная ссылка

Вручную:

```text
exp://ПУБЛИЧНЫЙ_IP:8081
```

Без `http://`, без лишнего слэша, **тот же IP**, что в шаге 5.  
Попробуй **Reload** в Expo Go (встряхни телефон → Reload).

---

## Шаг 7 — обход без диагностики сети

Если надоело ковырять порты:

```bash
cd /root/.openclaw/workspace/sophia-os
npm run start:tunnel
```

Открой в Expo Go **URL из вывода** терминала (иногда нужен `npx expo login`).

---

## Шаг 8 — логи Metro

В терминале, где крутится Metro, при попытке зайти с телефона должны появляться **запросы** / ошибки. Если **тишина** при открытии Safari `http://IP:8081` — пакет **не доходит** до процесса (фаервол, не тот IP, не тот порт).

---

## Краткий чеклист

1. [ ] `npm run start:lan` запущен и не упал  
2. [ ] `ss` показывает `0.0.0.0:8081`  
3. [ ] `curl http://127.0.0.1:8081` с сервера отвечает  
4. [ ] `ufw` не режет 8081 (или выключен)  
5. [ ] Safari на iPhone открывает `http://IP:8081`  
6. [ ] В Expo Go: `exp://IP:8081`  
7. [ ] Запасной план: `npm run start:tunnel`  

Автоскрипт: **`npm run check:mobile`**.
