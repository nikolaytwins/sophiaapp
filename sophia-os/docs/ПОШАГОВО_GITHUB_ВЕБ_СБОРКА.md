# Пошагово: как получить веб-версию приложения через GitHub (без программирования)

Эта инструкция для того случая, когда код уже лежит в проекте (как у тебя в Cursor), а ты хочешь **один раз настроить** и дальше нажимать кнопки.

**Зачем это нужно:** «собрать веб» — это тяжёлая операция. На слабом сервере (Selectel) она может **положить машину**. GitHub **сам** собирает проект на своих компьютерах — твоему серверу ничего не делается.

---

## Шаг 0. Что у тебя должно быть

1. **Аккаунт на GitHub** — сайт [github.com](https://github.com), регистрация по почте (бесплатно).
2. **Код проекта** где-то сохранён и ты можешь его **отправить на GitHub** (через Cursor, или через программу «GitHub Desktop» — см. ниже).

Если репозитория на GitHub ещё нет — сначала нужно **создать репозиторий** и **залить туда папку с проектом**. Это один раз; если не хочешь сам — попроси того, кто настраивал Cursor/сервер, сделать «push в GitHub».

---

## Шаг 1. Зайти в свой репозиторий на GitHub

1. Открой [github.com](https://github.com) и войди в аккаунт.
2. Нажми на свой **аватар** (правый верхний угол) → **Your repositories** (твои репозитории).
3. Открой **тот репозиторий**, куда залит этот проект (workspace с папкой `sophia-os` и файлом `.github/workflows/expo-web.yml`).

Если не знаешь название — в списке ищи репозиторий, в котором есть папка **sophia-os**.

---

## Шаг 2. Проверить, что файл workflow на месте

1. В репозитории нажми на папку **`.github`** (с точкой в начале).
2. Зайди в **`workflows`**.
3. Должен быть файл **`expo-web.yml`**.

Если папки `.github` нет — значит, код на GitHub **не обновляли** после того, как этот файл добавили. Нужно **снова залить проект** с твоего компьютера/сервера (тот, кто делает `git push`).

---

## Шаг 3. Запустить сборку вручную

1. В **верхней полоске** репозитория найди вкладку **Actions** (Действия) и нажми.
2. Слева в списке workflows выбери **Sophia OS — web export** (или похожее название из файла).
3. Справа сверху будет кнопка **Run workflow** (Запустить workflow).
4. Нажми **Run workflow** ещё раз в выпадающем окне (ветку оставь **main** или **master** — как у тебя в репозитории).
5. Подожди **несколько минут** (обычно 3–10). Страница сама обновится.

**Как понять, что всё ок:**

- У последнего запуска слева стоит **зелёная галочка** ✅ — успех.
- Красный крестик ❌ — ошибка; тогда нужен человек с опытом, чтобы открыть этот запуск и прочитать красный текст внизу (лог).

---

## Шаг 4. Скачать готовый сайт (архив)

1. Нажми на **успешный** запуск (зелёная галочка).
2. Прокрути вниз до секции **Artifacts** (Артефакты).
3. Там будет файл **`sophia-os-web-dist`** (или с похожим именем).
4. Нажми на него — начнётся **скачивание zip-архива** на твой компьютер.

Это и есть **готовая веб-версия**: внутри архива папка с файлами сайта (их потом выкладывают на домен).

---

## Шаг 5. Что делать с архивом дальше

Ты **не обязан** сам настраивать сервер. Варианты:

1. **Отправить архив или папку** тому, кто администрирует **app.twinlabs.ru** или хостинг, с текстом: «это статический сайт, нужно разложить в корень сайта для SPA».
2. Если пользуешься **Netlify / Vercel / Cloudflare Pages** — там обычно «перетащить папку» или подключить GitHub; это уже отдельная инструкция у них.

Главное: **архив из GitHub — это готовые файлы**, их не нужно «компилировать» на твоём Selectel.

---

## Если репозиторий только папка sophia-os (без родительской обёртки)

Иногда проект в GitHub лежит **так**, что в корне сразу `package.json`, а не папка `sophia-os`. Тогда файл **`.github/workflows/expo-web.yml`** из этой инструкции **не подойдёт без правок** — пути в нём рассчитаны на монорепо. Напиши тому, кто настраивает Git: «нужно поправить workflow под один каталог sophia-os» — или используй документ [GITHUB_WEB_BUILD.md](./GITHUB_WEB_BUILD.md), раздел в конце.

---

## Короткая шпаргалка

| Действие | Где |
|----------|-----|
| Запустить сборку | GitHub → репозиторий → **Actions** → **Sophia OS — web export** → **Run workflow** |
| Скачать результат | Успешный запуск → внизу **Artifacts** → скачать zip |
| Не грузить Selectel | Сборка идёт на **GitHub**, не на твоём VPS |

Если на каком-то шаге **нет вкладки Actions** или **нет Run workflow** — чаще всего репозиторий **пустой**, **приватный с ограничениями**, или код **ещё не запушен** с файлом `.github/workflows/`. Тогда сначала нужно **обновить код на GitHub** из проекта, где лежит этот файл.
