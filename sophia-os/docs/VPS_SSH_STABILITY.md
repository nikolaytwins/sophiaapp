# Почему «отваливается» SSH на VPS после Expo / сборки веба

## Что происходит по симптомам

- Через ~минуту после тяжёлой команды (Metro, `expo export`, большой `npm` build) **пропадает SSH**, переподключение из Cursor не помогает.
- Помогает только **программная перезагрузка ВМ в панели Selectel**.

Это очень похоже не на «баг SSH», а на **исчерпание ресурсов сервера**: ядро убивает процессы (**OOM killer**) или вся машина уходит в **disk thrashing / полный стоп** из‑за нехватки RAM. Тогда недоступны и SSH, и всё остальное, пока не перезагрузишь железо/ВМ.

**Expo / Metro / export веба** — памятоёмкие Node-процессы; на ВМ с **512 МБ–1 ГБ RAM** без swap одна сборка может положить хост.

## Что не делать на маленьком VPS (или делать только осознанно)

| Действие | Риск |
|----------|------|
| `npx expo export --platform web` | Высокий пик RAM + CPU |
| Долгий `expo start` с полным бандлом впервые | Высокий |
| Несколько тяжёлых `npm` параллельно | Высокий |
| Лёгкие вещи: `tsc --noEmit`, правки файлов, `git` | Обычно ок |

**Сборку веба и тяжёлый Metro** разумно гонять **на Mac**, в **GitHub Actions** (см. [GITHUB_WEB_BUILD.md](./GITHUB_WEB_BUILD.md)), или на отдельной машине с запасом RAM — не на том же дроплете, где ты живёшь по SSH.

## Что проверить после очередного зависания (когда снова зайдёшь)

```bash
free -h
sudo dmesg -T | tail -80 | grep -i -E 'oom|kill|out of memory'
df -h
```

Если в `dmesg` есть **Out of memory** / **Killed process** — это подтверждение OOM.

## Мягкие меры (не панацея)

1. **Swap** на ВМ (если ещё нет) — снижает шанс мгновенного убийства процессов, но при сильной нехватке всё равно будет тяжело.
2. **Лимит памяти Node** (не спасёт от всего, но снижает пики):  
   `export NODE_OPTIONS=--max-old-space-size=1024` перед командой (подставь разумное число под план).
3. **Апгрейд тарифа** Selectel — больше RAM проще всего стабилизирует поведение.

## Правило для агентов / Cursor в этом репо

Не запускать на **удалённом VPS по SSH** без явного согласия человека:

- `expo export` (веб),
- долгий первый бандл Metro,
- любые **долгие** тяжёлые сборки.

Писать код и проверять типы **`tsc`** — нормально. Спрашивать человека: «собрать веб локально на Mac или в CI?» — если нужна именно сборка.

---

Итог: **падение SSH после моих команд — не «сломался Expo»**, а с большой вероятностью **нагрузка на слабую ВМ**. Переносим тяжёлое с VPS, добавляем RAM/swap или осознанно запускаем редко.
